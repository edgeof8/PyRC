Debugging and Logging

This section details PyRC's logging mechanisms and provides guidance on how to use them for debugging and troubleshooting issues.

Logging Overview

PyRC provides comprehensive logging capabilities to help diagnose issues and understand client behavior. Logs are automatically created and stored in the logs/ directory.

  • Main application log: logs/pyrc_core.log
  • DCC transfer log: logs/dcc.log
  • Per-channel logging (optional): Configurable via pyterm_irc_config.ini.
  • Status window log: logs/client_status_messages.log

Configuring Logging

Logging behavior can be configured in the [Logging] section of pyterm_irc_config.ini, allowing control over log levels, file paths, and rotation settings.


[Logging]
log_enabled = true
log_file = pyrc_core.log
log_level = INFO
channel_log_enabled = false

Using Raw IRC Message Logging

The /rawlog [on|off|toggle] command allows you to display raw IRC messages directly in the Status window, which is invaluable for debugging protocol-level interactions.

Leveraging State Management for Debugging

The `StateManager` provides a centralized and persistent view of the client's runtime state. Understanding its structure and contents (e.g., in `state.json`) can be crucial for debugging unexpected client behavior or connection issues.

Key areas for debugging with `StateManager`:

  • ConnectionInfo: Verify server, port, nick, and authentication details.
  • ConnectionState: Track the client's connection lifecycle (e.g., DISCONNECTED, CONNECTING, REGISTERED).
  • Persistent state: Examine `state.json` for unexpected saved values or corruption.
