Troubleshooting

This section provides solutions to common issues encountered while using PyRC.

Connection Issues

If you're having trouble connecting to an IRC server:

  • Check Server Address and Port: Ensure the server address and port in `pyterm_irc_config.ini` are correct.
  • SSL/TLS: Verify if the server requires SSL and if the `ssl` option is set correctly in your configuration.
  • Firewall/Network: Ensure your firewall or network configuration is not blocking the connection.
  • Nickname in Use: If your chosen nickname is already in use, PyRC will attempt to find an alternative. You can change your default nick in `pyterm_irc_config.ini`.
  • SASL Authentication: If the server requires SASL, ensure `sasl_username` and `sasl_password` are correctly configured.

UI Display Problems

If the terminal UI is not rendering correctly:

  • Terminal Compatibility: PyRC uses the `curses` library. Ensure your terminal emulator supports `curses` (e.g., PuTTY, Alacritty, VSCode integrated terminal).
  • Windows-curses: On Windows, ensure `windows-curses` is installed (`pip install windows-curses`).
  • Terminal Size: Resizing your terminal might resolve some display glitches.

Scripting Errors

If your custom scripts are not working as expected:

  • Check Logs: Review `logs/pyrc_core.log` for any Python tracebacks or errors related to script execution.
  • Script Path: Ensure your script is placed in the `scripts/` directory and is not disabled in `pyterm_irc_config.ini`.
  • API Usage: Verify that you are using the `ScriptAPIHandler` methods correctly, especially `async` methods that require `await`.
  • Dependencies: Check for any missing Python dependencies required by your script.

DCC Transfer Issues

For problems with DCC file transfers:

  • Firewall: DCC transfers often require direct connections. Ensure your firewall allows incoming connections on the configured DCC port range.
  • Advertised IP: If you are behind NAT, consider setting `dcc_advertised_ip` in `pyterm_irc_config.ini` to your public IP.
  • Port Range: Ensure the `port_range_start` and `port_range_end` are open and not in use by other applications.
  • Blocked Extensions/File Size: Check your `dcc_blocked_extensions` and `max_file_size` settings.
  • DCC Log: Review `logs/dcc.log` for specific transfer errors.

General Debugging Steps

  • Enable Raw Logging: Use `/rawlog on` to see raw IRC messages, which can help pinpoint communication issues.
  • Check `state.json`:** Examine `state.json` in the `config/` directory to understand the client's persistent state.
  • Restart PyRC: A fresh restart can sometimes resolve transient issues.
  • Update PyRC: Ensure you are running the latest version of PyRC.
  • Open an Issue: If you cannot resolve the issue, open a detailed issue on the GitHub repository, including logs and steps to reproduce.
